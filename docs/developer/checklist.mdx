## Basics

|   | Checklist Item 
|---|:--------------|
| <input type="checkbox" /> | 1. **Semantic HTML**: Use proper HTML elements for their intended purpose (e.g., `<button>` for buttons, `<header>` for headers). |
| <input type="checkbox" /> | 2. **Keyboard Navigation**: Ensure all interactive elements are accessible via keyboard. |
| <input type="checkbox" /> | 3. **ARIA Roles**: Use ARIA roles sparingly and only when necessary to enhance accessibility. |
| <input type="checkbox" /> | 4. **Color Contrast**: Verify sufficient contrast between text and background colors. |
| <input type="checkbox" /> | 5. **Alt Text**: Provide meaningful alt text for images. |
| <input type="checkbox" /> | 6. **Focus Management**: Ensure focus is managed correctly, especially for dynamic content. |
| <input type="checkbox" /> | 7. **Error Handling**: Make error messages clear and accessible. |
| <input type="checkbox" /> | 8. **Responsive Design**: Ensure content adapts well to different screen sizes and orientations. |
| <input type="checkbox" /> | 9. **Testing**: Test with screen readers and other assistive technologies. |
| <input type="checkbox" /> | 10. **A11y ESLint**: Enable and use `eslint-plugin-jsx-a11y` to catch accessibility issues during development. See [Tools](/tools) for setup instructions. |
