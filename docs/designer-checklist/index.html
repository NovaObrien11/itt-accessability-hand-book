Designer Accessibility Checklist

Design Phase​

Checklist ItemResource
1.3.3 Sensory-Agnostic Instructions - Ensure instructions are not relying on a single sensory ability.WCAG 2.2 - Sensory Characteristics
1.4.1 Information Color - Use something besides color alone to convey information.WCAG 2.2 - Use of Color
1.4.3 Color Contrast - Ensure color contrast ratios meet minimum requirements.WCAG 2.2 - Contrast Minimum
1.4.5 Text as Image - Use live text whenever possible instead of images of text (e.g., logo).WCAG 2.2 - Images of Text
1.4.8 Visual Presentation - Width is no more than 80 characters or glyphs (40 if CJK).WCAG 2.2 - Visual Presentation
1.4.11 Color Contrast (Non-text) - All meaningful non-text content (e.g., buttons, icons) should have a minimum 3:1 color contrast ratio.WCAG 2.2 - Non-text Contrast
2.3.1 Flashing Content - Avoid content that flashes more than three times per second.WCAG 2.2 - Three Flashes or Below Threshold
2.4.3 Content Order - Users must be able to navigate in a logical order that preserves meaning.WCAG 2.2 - Focus Order
2.4.4 Meaningful Anchor Text - The purpose of each link should be clear from its anchor text.WCAG 2.2 - Link Purpose in Context
2.4.5 Discoverable Content - Multiple ways to find different pages and content.WCAG 2.2 - Multiple Ways
2.4.6 Meaningful Wording - Headings and labels must be clear and descriptive.WCAG 2.2 - Headings and Labels
2.4.7 Focus Visibility - UI controls that receive keyboard focus should have a visible indicator.WCAG 2.2 - Focus Visible
2.4.11 Discoverable Focus - Focused elements must not be hidden or blocked by other content.WCAG 2.2 - Focus Not Obscured Minimum
2.5.4 Motion Control - Provide alternatives for motion-activated functions and allow disabling motion activation.WCAG 2.2 - Motion Actuation
2.5.8 Target Size - Interactive elements must be at least 24x24 CSS pixels.WCAG 2.2 - Target Size Minimum
3.2.1 Context Changes - Context must not change just because an item receives focus.WCAG 2.2 - On Focus
3.2.2 UI Recognition - UI changes should not occur automatically due to user input without warning.WCAG 2.2 - On Input
3.2.3 System Consistency - Navigation and layout must be consistent across all pages.WCAG 2.2 - Consistent Navigation
3.2.4 Consistent Design Patterns - Components with the same function are consistently identified.WCAG 2.2 - Consistent Identification
3.2.6 Discoverable Documentation - Help options (support, contact, chatbots) should be consistent and predictable.WCAG 2.2 - Consistent Help
3.3.1 Form Errors - Make form errors easy to identify, understand, and correct.WCAG 2.2 - Error Identification
3.3.2 Form Instructions - Provide clear visual and programmatic labels or instructions for input fields.WCAG 2.2 - Labels or Instructions
3.3.3 Error Fixes - If an input error is detected, provide suggestions for correcting it.WCAG 2.2 - Error Suggestion

Development Phase​

Checklist ItemResource
1.1.1 Alt Text - All images and non-text content must have a text alternative.WCAG 2.2 - Non-text Content
1.2.5 Audio Descriptions - Provide both transcripts and audio descriptions for audio content.WCAG 2.2 - Audio Description Prerecorded
1.4.2 Audio Controls - Provide controls to pause, stop, or mute audio.WCAG 2.2 - Audio Control
1.4.13 Content Accessibility - Additional content (pop-ups, submenus) can be dismissed or remain visible as desired.WCAG 2.2 - Content on Hover or Focus
2.1.2 Keyboard Navigation - Keyboard-only users must be able to navigate without getting stuck.WCAG 2.2 - No Keyboard Trap
2.2.2 Motion Within Content - Users must be able to pause, stop, or hide blinking, scrolling, or moving content.WCAG 2.2 - Enough Time
3.3.4 Clear Recognition, Diagnoses, and Recovery - For legal/financial pages, submissions must be reversible, errors correctable, and confirmation available.WCAG 2.2 - Error Prevention Legal Financial Data
3.3.8 Clear Authentication - Users can authenticate without relying solely on memory/cognitive tests; alternatives or assistance are provided.WCAG 2.2 - Accessible Authentication Minimum

Testing Phase​

Checklist ItemResource
1.4.4 Text Scale - Text must be resizable up to 200% without loss of content or functionality.WCAG 2.2 - Resize Text
1.4.12 Adjustable Text Spacing - Text spacing can be adjusted without causing a poor experience.WCAG 2.2 - Text Spacing
