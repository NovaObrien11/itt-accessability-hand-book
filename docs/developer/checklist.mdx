> "The best way to ensure accessibility is to bake it into your workflow, not bolt it on at the end."

## <Toolbox size={32} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Tooling & Setup (Do This First)

Before writing code, ensure your environment is set up to catch issues early.

|   | Checklist Item |
|---|:--------------|
| <Square size={24} /> | **Install ESLint Plugin**: Add `eslint-plugin-jsx-a11y` to your project. <br/>`npm install eslint-plugin-jsx-a11y --save-dev` |
| <Square size={24} /> | **Configure VS Code**: Install the **axe Accessibility Linter** extension for real-time feedback. |
| <Square size={24} /> | **CI/CD Gate**: Ensure `axe-core` or `pa11y` runs in your build pipeline. Fail the build on new violations. |

## <Laptop size={32} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Implementation Checklist

### <Code size={28} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Semantic HTML & Structure
|   | Checklist Item |
|---|:--------------|
| <Square size={24} /> | **Landmarks**: Use `<main>`, `<nav>`, `<header>`, `<footer>`, and `<aside>` to define page structure. |
| <Square size={24} /> | **Headings**: Use `<h1>` through `<h6>` in logical order. No skipped levels (e.g., don't jump from `h1` to `h3`). |
| <Square size={24} /> | **Buttons vs. Links**: Use `<button>` for actions (submit, toggle) and `<a>` for navigation. |
| <Square size={24} /> | **Lists**: Use `<ul>`, `<ol>`, and `<dl>` for list content, not `<div>` soup. |

### <Keyboard size={28} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Keyboard & Focus
|   | Checklist Item |
|---|:--------------|
| <Square size={24} /> | **Visible Focus**: Never set `outline: none` without providing a visible alternative style. |
| <Square size={24} /> | **Tab Order**: Ensure tab order follows the visual flow. Avoid positive `tabindex` values (e.g., `tabindex="1"`). |
| <Square size={24} /> | **Skip Links**: Include a "Skip to Main Content" link as the first focusable element. |
| <Square size={24} /> | **Focus Traps**: Modals and drawers must trap focus while open and return focus to the trigger when closed. |

### <Image size={28} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Images & Media
|   | Checklist Item |
|---|:--------------|
| <Square size={24} /> | **Alt Text**: All `<img>` tags have an `alt` attribute. Use `alt=""` for decorative images. |
| <Square size={24} /> | **No Text in Images**: Avoid baking text into images. If unavoidable, ensure the `alt` text contains the full message. |
| <Square size={24} /> | **Captions**: Ensure all video content has closed captions. |

### <CursorClick size={28} style={{verticalAlign: 'middle', marginRight: '10px'}} /> ARIA (Accessible Rich Internet Applications)
> **First Rule of ARIA**: Don't use ARIA if a native HTML element will do.

|   | Checklist Item |
|---|:--------------|
| <Square size={24} /> | **Valid Roles**: Use valid ARIA roles (e.g., `role="dialog"`, `role="alert"`). |
| <Square size={24} /> | **State Management**: Update `aria-expanded`, `aria-hidden`, and `aria-selected` dynamically via JavaScript. |
| <Square size={24} /> | **Labeling**: Use `aria-label` or `aria-labelledby` for interactive elements without visible text (e.g., icon buttons). |

## <RocketLaunch size={32} style={{verticalAlign: 'middle', marginRight: '10px'}} /> Definition of Done (DoD)

A feature is **NOT** complete until:
1.  <Square size={20} style={{verticalAlign: 'text-bottom', marginRight: '8px'}} /> No linting errors from `eslint-plugin-jsx-a11y`.
2.  <Square size={20} style={{verticalAlign: 'text-bottom', marginRight: '8px'}} /> Automated tests (axe-core) pass.
3.  <Square size={20} style={{verticalAlign: 'text-bottom', marginRight: '8px'}} /> Manual keyboard navigation test passes (Tab through the entire flow).
4.  <Square size={20} style={{verticalAlign: 'text-bottom', marginRight: '8px'}} /> Screen reader verification (VoiceOver/NVDA) confirms usability.
