## ARIA Resources

### Essential References

- **[W3C ARIA Authoring Practices Guide (APG)](https://www.w3.org/WAI/ARIA/apg/)** - The official W3C guide for building accessible web components with ARIA. Includes design patterns, examples, and fundamental practices for implementing ARIA correctly.
  - [Design Patterns](https://www.w3.org/WAI/ARIA/apg/patterns/) - Accessible component patterns (tabs, modals, comboboxes, etc.)
  - [ARIA Landmarks](https://www.w3.org/WAI/ARIA/apg/practices/landmark-regions/) - Using landmarks for page structure
  - [Accessible Names and Descriptions](https://www.w3.org/WAI/ARIA/apg/practices/names-and-descriptions/) - How to provide proper accessible names

### Additional Resources

- [WAI-ARIA: For Average Web Developers](https://www.davidmacd.com/blog/wai-aria-accessbility-for-average-web-developers.html) - Beginner-friendly ARIA introduction
- [MDN: ARIA Overview](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA) - Comprehensive ARIA documentation
- [a11y Support (ARIA tables by feature)](https://a11ysupport.io/) - Browser and screen reader support tables
- [Powermapper: ARIA Screen Reader Tests](https://www.powermapper.com/tests/screen-readers/aria/) - Real-world ARIA testing results
- [HTML5 Accessibility](https://html5accessibility.com/) - HTML5 element accessibility support
